# Дизайн-документ: Условия переходов между эрами

Этот документ фиксирует точные progression gates: какие условия нужны, чтобы
поселение могло перейти из одной эры в следующую.

Дизайн-описание самих эр находится в [eras_overview.md](eras_overview.md).
Поперечные цепочки зданий и апгрейдов находятся в
[building_progression.md](building_progression.md).

## 1. Принципы gates

Переход между эрами не должен быть автоматическим. Игрок должен доказать, что
поселение освоило ключевую петлю текущей эры:

* построило центральные здания;
* закрыло базовые бытовые потребности;
* освоило нужные материалы;
* открыло инструменты или профессии;
* подготовило логистику для следующего масштаба.

## 2. Источник правды

Текущая реализация правил находится в `SettlementState.can_advance_to()`:

```text
game/features/settlement/domain/settlement_state.gd
```

Этот документ должен оставаться рядом с кодом по смыслу: если меняется gate в
коде, нужно обновить соответствующий раздел здесь.

## 3. Реализованные gates

### 3.1. `TENT -> EARTH`

Текущая эра должна быть `TENT`.

Нужны инструменты:

* `axe`;
* `hand_saw`;
* `shovel`;
* `bucket`.

Дизайн-обоснование: инструменты покупаются через расширенную торговлю
палаточного рынка. Поэтому костер 3-го уровня, рынок и базовая экономика
являются естественными предпосылками, но не дублируются отдельными условиями
перехода.

### 3.2. `EARTH -> CLAY`

Текущая эра должна быть `EARTH`.

Нужны здания:

* `earth_assembly`;
* `smithy`;
* `earth_market`;
* `toilet_earth_lvl3`.

Нужны ресурсы и экономика:

* `housing_slots >= population`;
* `clay >= 5`;
* `money >= 5`;
* `trade_sales >= 3`.

Нужны инструменты:

* `hoe`.

Дизайн-обоснование: поселение должно иметь собрание, кузницу, рынок,
полностью закрытое жилье, продвинутую гигиену и первый опыт торговли.

### 3.3. `CLAY -> WOOD`

Текущая эра должна быть `CLAY`.

Нужны здания:

* `clay_lodge`;
* `clay_market`;
* `toilet_clay_lvl3`.

Нужны ресурсы и экономика:

* `water >= population`;
* `logs >= 10`;
* `money >= 10`.

Дизайн-обоснование: глиняное поселение должно стабилизировать воду, торговлю,
гигиену и накопить запас бревен для перехода к деревообработке.

### 3.4. `WOOD -> STONE`

Текущая эра должна быть `WOOD`.

Нужны здания:

* `wood_town_hall`;
* `wood_market`;
* `house_lvl3`;
* `toilet_wood_lvl3`.

Нужны ресурсы и экономика:

* `money >= 15`.

Нужны инструменты:

* `pickaxe`.

Дизайн-обоснование: деревянный город должен иметь ратушу, рынок, развитое
жилье, гигиену и инструмент для камня.

### 3.5. `STONE -> BRICK`

Текущая эра должна быть `STONE`.

Нужны здания:

* `stone_prefecture`;
* `stone_market`;
* `masonry_workshop`;
* `toilet_stone_lvl3`.

Нужны ресурсы и экономика:

* `stone >= 20`;
* `money >= 20`.

Дизайн-обоснование: каменный город должен освоить управление, рынок, каменную
обработку и иметь запас материала для кирпичного строительного рывка.

## 4. Планируемые gates

### 4.1. `BRICK -> ECO`

Пока не реализовано в доменной модели.

Предполагаемые темы gate:

* загрязнение или износ среды достигли уровня, требующего ответа;
* построены кирпичная администрация и крупный рынок;
* есть производство стекла, топлива и базовой электроэнергии;
* открыты первые экологические технологии;
* Wellbeing не провален из-за плотности и промышленности.

### 4.2. `ECO -> INDUSTRIAL`

Пока не реализовано в доменной модели.

Предполагаемые темы gate:

* поселение умеет балансировать энергию, отходы и транспорт;
* открыты материалы для крупной инфраструктуры;
* есть база для железных дорог или магистральной логистики;
* игрок готов перейти от симуляции жителей к макро-управлению.

## 5. Что не должно попадать в gates

Не каждое важное здание нужно добавлять как жесткое условие перехода.

Хороший gate проверяет освоение петли. Плохой gate превращает прогрессию в
чеклист без смысла.

Например, для `TENT -> EARTH` не нужно отдельно требовать костер 3-го уровня и
палаточный рынок, если нужные инструменты уже покупаются только через этот путь.
Такое требование дублировало бы одну и ту же предпосылку.
