# Дизайн-документ: Область деревни (Village Territory)

Этот документ описывает механику территориальных границ поселения: где можно
строить, как область расширяется и как отображаются границы.

Связанные документы:

- [building_progression.md](building_progression.md) — цепочки зданий, уровни, апгрейды;
- [tent_era_survival.md](tent_era_survival.md) — стартовый сценарий, Главный костёр, входная табличка;
- [storage_warehouses.md](storage_warehouses.md) — склады под открытым небом;
- [architecture.md](../docs/architecture.md) — фичи, слои, границы владения.

---

## 1. Концепция

У поселения есть **область деревни** — территория, в пределах которой разрешено
строительство. Область формируется поэтапно:

1. **Склад под открытым небом** можно построить где угодно — он не требует
   области деревни и сам её не создаёт.
2. **Главный костёр** (campfire) — первое здание, создающее область деревни.
   Вокруг него образуется круг допустимого радиуса строительства.
3. **Жилые дома**, построенные в радиусе костра, расширяют область деревни:
   каждый дом добавляет свой радиус к общей территории.
4. Внутри объединённой области можно строить любые здания, кроме склада под
   открытым небом (который можно строить и за пределами области).
5. **Нельзя строить в области чужой деревни.** Если на карте присутствует
   враждебное или нейтральное поселение со своей областью, пересечение
   территорий блокирует размещение чертежа.

Таким образом, костёр — это якорь поселения, а дома — инструмент расширения.
Игрок не может расползаться по карте произвольно: рост деревни органичен и
требует постройки жилья.

## 2. Правила строительства

### 2.1. Иерархия разрешений

| Тип здания | Требует область деревни? | Расширяет область? |
| :--- | :---: | :---: |
| Склад под открытым небом | Нет | Нет |
| Главный костёр | Нет (сам создаёт) | Да (якорь) |
| Костёр для готовки | Да | Нет |
| Жилые дома (палатки, землянки, дома и т.д.) | Да | Да |
| Производственные здания | Да | Нет |
| Рынки, туалеты, места отдыха | Да | Нет |
| Сборщик росы / склад воды | Да | Нет |

### 2.2. Условие размещения

При попытке разместить чертёж проверяются два условия:

1. **Точка находится внутри области деревни** (для зданий, требующих область).
2. **Точка не находится внутри области чужой деревни** (для всех зданий).

Если хотя бы одно условие не выполнено, размещение блокируется, а UI
показывает причину: «Вне области деревни» или «Территория чужого поселения».

### 2.3. Радиусы

- **Радиус костра** — базовый радиус области, создаваемой Главным костром.
  Значение настраивается по уровню костра: чем выше уровень, тем больше
  начальный радиус застройки.
- **Радиус дома** — радиус, который каждый жилой дом добавляет к области.
  Значение может зависеть от уровня дома (большие дома расширяют территорию
  сильнее).
- **Область деревни** — объединение всех кругов: костёр + каждый жилой дом.
  Область хранится как множество клеток сетки (grid cells), входящих хотя бы
  в один круг.

### 2.4. Снос и область

При сносе жилого дома его радиус удаляется из объединённой области. Если в
результате какая-то часть территории выходит за пределы области, здания,
оставшиеся в этой части, **не разрушаются автоматически**, но новое
строительство там становится невозможным до восстановления области.

При сносе Главного костра область деревни **полностью аннулируется**, если в
поселении нет другого источника области (например, второго костра в
мульти-поселенческом сценарии). Все жилые дома остаются, но перестают
расширять территорию, пока не будет построен новый костёр.

---

## 3. Чужие деревни

На карте могут присутствовать другие поселения со своими областями. Правила:

- Каждое поселение имеет независимую область, рассчитанную по тем же правилам
  (костёр + дома).
- Игрок не может строить в области чужого поселения.
- Чужое поселение не может строить в области игрока.
- Области могут **соприкасаться**, но не **пересекаться**: если радиус нового
  здания игрока накладывается на чужую область, размещение блокируется.
- Дипломатия, торговля и взаимодействие между областями — тема будущего
  дизайна; в текущей механике области только ограничивают строительство.

## 4. Визуализация границ

Граница области деревни отображается **малыми столбиками** (boundary posts),
которые автоматически размещаются по периметру зданий, формирующих область:

- Столбики ставятся вдоль внешних граней footprint-ов зданий, образующих
  границу объединённой области.
- Столбики не являются зданиями: они не занимают строительных клеток, не
  имеют здоровья и не отображаются в меню строительства.
- При расширении области (постройка нового дома) столбики автоматически
  пересчитываются.
- При сносе здания столбики также пересчитываются.
- Цвет столбиков: нейтральный (дерево/камень) для своей деревни. При
  наведении курсора в режиме строительства столбики подсвечиваются, показывая
  текущую границу.

## 5. Стартовый сценарий

В палаточной эре последовательность:

1. Группа прибывает к входной табличке.
2. Игрок строит **склад под открытым небом** — где угодно, без ограничений.
3. Игрок строит **Главный костёр 1-го уровня** — где угодно, без ограничений.
   С момента завершения строительства костра вокруг него образуется область
   деревни с базовым радиусом.
4. В радиусе костра игрок строит **временную палатку** или **жилую палатку**.
   Палатка расширяет область.
5. В расширенной области строятся костёр для готовки, сборщик росы, туалет и
   т.д.

До постройки костра игрок может строить **только** склад под открытым небом.
Это мотивирует быстро завершить строительство костра и не откладывать
развитие лагеря.

## 6. Архитектура реализации

### 6.1. Домен

`game/features/buildings/domain/village_territory.gd`:

- `VillageTerritory` — типизированная запись: центр (костёр), радиус костра,
  список жилых домов с их позициями и радиусами.
- Детерминированные функции: `is_inside(cell: Vector2i) -> bool`,
  `is_inside_foreign(cell: Vector2i) -> bool`.
- Расчёт объединённой области — множество клеток сетки.

### 6.2. Приложение

`game/features/buildings/application/village_territory_service.gd`:

- Читает `BuildingRegistry` для отслеживания построенных костров и жилых
  домов.
- Пересчитывает область при постройке, сносе, апгрейде зданий.
- Предоставляет `can_place(building_type: StringName, cell: Vector2i) -> bool`
  и `placement_reason(...) -> String` для UI.
- Хранит список чужих деревень (для будущих сценариев с NPC-поселениями).

### 6.3. Интеграция

- `_can_place` в `settlement_game.gd` дополняется вызовом
  `village_territory_service.can_place(...)`.
- `BuildingRegistry` эмитит сигнал при добавлении/удалении записи; сервис
  подписывается и пересчитывает область.
- `building_catalog.gd` получает флаг `requires_village_area: bool` и
  `expands_village_area: bool` для каждого определения здания.

### 6.4. Презентация

`game/features/buildings/presentation/village_boundary_markers.gd`:

- `Node3D`, который подписывается на изменения области и обновляет столбики.
- Столбики — простые мешевые инстансы, не влияющие на навигацию и коллизию.
