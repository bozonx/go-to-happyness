# Дизайн-документ: Область деревни (Village Territory)

Этот документ описывает механику территориальных границ поселения: где можно
строить, как область расширяется и как отображаются границы.

Связанные документы:

- [building_progression.md](building_progression.md) — цепочки зданий, уровни, апгрейды;
- [tent_era_survival.md](tent_era_survival.md) — стартовый сценарий, Главный костёр, входная табличка;
- [storage_warehouses.md](storage_warehouses.md) — склады под открытым небом;
- [architecture.md](../docs/architecture.md) — фичи, слои, границы владения.

---

## 1. Концепция

У поселения есть **область деревни** — территория, в пределах которой разрешено
строительство. Область формируется поэтапно:

1. **Склад под открытым небом** можно построить где угодно — он не требует
   области деревни и сам её не создаёт.
2. **Главный костёр** (campfire) — первое здание, создающее область деревни.
   Вокруг него образуется базовый радиус застройки.
3. **Жилые дома** и **столбы** (boundary posts), построенные в области
   деревни, расширяют её: каждый дом и каждый столб добавляет свой радиус к
   общей территории.
4. Внутри объединённой области можно строить любые здания, кроме склада под
   открытым небом (который можно строить и за пределами области).
5. **Нельзя строить в области чужой деревни.** Если на карте присутствует
   враждебное или нейтральное поселение со своей областью, пересечение
   территорий блокирует размещение чертежа.

Таким образом, костёр — это якорь поселения, а дома и столбы — инструменты
расширения. Игрок не может расползаться по карте произвольно: рост деревни
органичен и требует постройки жилья или установки столбов.

### 1.1. Устаревшая система радиуса

**Старая система**, где вокруг костра давался фиксированный радиус
строительства без понятия области деревни, **полностью убирается**. Вместо
неё вводится область деревни как множество клеток сетки, формируемое костром,
домами и столбами. Все проверки размещения зданий теперь идут через
`village_territory_service`, а не через простой радиус от костра.

## 2. Правила строительства

### 2.1. Иерархия разрешений

| Тип здания | Требует область деревни? | Расширяет область? |
| :--- | :---: | :---: |
| Склад под открытым небом | Нет | Нет |
| Главный костёр | Нет (сам создаёт) | Да (якорь) |
| Столб (boundary post) | Да | Да |
| Костёр для готовки | Да | Нет |
| Жилые дома (палатки, землянки, дома и т.д.) | Да | Да |
| Производственные здания | Да | Нет |
| Рынки, туалеты, места отдыха | Да | Нет |
| Сборщик росы / склад воды | Да | Нет |

### 2.1.1. Столб (boundary post)

**Столб** — дешёвое строение из меню построек, предназначенное для
расширения области деревни без постройки полноценного дома.

- **Стоимость:** 2 ветки.
- **Footprint:** 1×1 клетка.
- **Не требует исследования** — доступен с момента постройки первого костра.
- **Расширяет область:** вокруг столба образуется малый радиус, который
  добавляется к объединённой области деревни.
- **Можно строить только внутри существующей области** — столб нельзя
  поставить за пределами деревни, чтобы «телепортировать» территорию.
  Столб расширяет границу, но не создаёт новую область в произвольной точке.
- **Можно сносить** — при сносе столба его радиус удаляется из области.
- **Не является жильём** — не предоставляет спальных мест, не заселяется
  юнитами.
- **Визуально** — небольшой деревянный столбик, который одновременно служит
  и маркером границы, и инструментом расширения.

### 2.2. Блокировка до постройки костра

**До постройки первого Главного костра** в меню построек доступны только
два типа зданий:

- **Склад под открытым небом** — можно строить где угодно.
- **Главный костёр 1-го уровня** — можно строить где угодно.

Все остальные здания **заблокированы** и отображаются в меню как
недоступные с причиной «Требуется костёр». Это включает палатки, костёр для
готовки, сборщик росы, туалеты, столбы и всё остальное.

Как только первый костёр завершён, область деревни формируется и все
остальные здания разблокируются (с учётом исследований и эры).

### 2.3. Условие размещения

При попытке разместить чертёж проверяются два условия:

1. **Точка находится внутри области деревни** (для зданий, требующих область).
2. **Точка не находится внутри области чужой деревни** (для всех зданий).

Если хотя бы одно условие не выполнено, размещение блокируется, а UI
показывает причину: «Вне области деревни» или «Территория чужого поселения».

### 2.4. Радиусы

- **Радиус костра** — базовый радиус области, создаваемой Главным костром.
  Значение настраивается по уровню костра: чем выше уровень, тем больше
  начальный радиус застройки.
- **Радиус дома** — радиус, который каждый жилой дом добавляет к области.
  Значение может зависеть от уровня дома (большие дома расширяют территорию
  сильнее).
- **Радиус столба** — малый радиус, который каждый столб добавляет к области.
  Значение фиксировано и меньше радиуса дома; столб — это дешёвый способ
  немного сдвинуть границу.
- **Область деревни** — объединение всех кругов: костёр + каждый жилой дом +
  каждый столб. Область хранится как множество клеток сетки (grid cells),
  входящих хотя бы в один круг.

### 2.5. Снос и область

При сносе жилого дома его радиус удаляется из объединённой области. Если в
результате какая-то часть территории выходит за пределы области, здания,
оставшиеся в этой части, **не разрушаются автоматически**, но новое
строительство там становится невозможным до восстановления области.

При сносе Главного костра область деревни **полностью аннулируется**, если в
поселении нет другого костра (см. раздел 3 «Количество поселений»). Все
жилые дома и столбы остаются, но перестают расширять территорию, пока не
будет построен новый костёр.

## 3. Количество поселений и костров

### 3.1. Одно поселение на старте

Изначально игрок может построить **только один Главный костёр** — одно
поселение. Это ограничение действует всю Палаточную эру.

### 3.2. Новые костры при открытии новых эр

При переходе в новую эру игрок получает возможность построить **дополнительный
костёр** — то есть основать второе поселение. Каждая новая эра увеличивает
лимит костров на 1:

| Эра | Лимит костров (поселений) |
| :--- | :---: |
| Палаточная | 1 |
| Земляная | 2 |
| Глиняная | 3 |
| Деревянная | 4 |
| Каменная | 5 |
| Кирпичная | 6 |

Новый костёр можно поставить только за пределами существующих областей
деревень — он создаёт новое независимое поселение со своей областью.

### 3.3. Чужие деревни

На карте могут присутствовать NPC-поселения со своими областями. Правила:

- Каждое поселение имеет независимую область, рассчитанную по тем же правилам
  (костёр + дома + столбы).
- Игрок не может строить в области чужого поселения.
- Чужое поселение не может строить в области игрока.
- Области могут **соприкасаться**, но не **пересекаться**: если радиус нового
  здания или столба игрока накладывается на чужую область, размещение
  блокируется.
- Дипломатия, торговля и взаимодействие между областями — тема будущего
  дизайна; в текущей механике области только ограничивают строительство.

## 4. Визуализация границ

Граница области деревни отображается двумя способами:

### 4.1. Автоматические маркеры

Вдоль внешнего периметра объединённой области автоматически размещаются
**малые маркеры** — визуальные столбики, не являющиеся зданиями:

- Маркеры ставятся по границе объединённой области (по контуру множества
  клеток).
- Маркеры не занимают строительных клеток, не имеют здоровья и не
  отображаются в меню строительства.
- При расширении или сужении области маркеры автоматически пересчитываются.
- Цвет маркеров: нейтральный (дерево/камень) для своей деревни. При
  наведении курсора в режиме строительства маркеры подсвечиваются, показывая
  текущую границу.

### 4.2. Столбы как расширяемые маркеры

**Столбы** (boundary posts) из меню построек — это одновременно и
инструмент расширения области, и видимый элемент границы:

- Столб стоит 2 ветки и занимает 1×1 клетку.
- Столб — это настоящее здание (с записью в `BuildingRegistry`), его можно
  сносить.
- Визуально столб выглядит как деревянный колышек высотой ~0.5 м.
- Столбы позволяют игроку вручную управлять формой границы, добавляя радиус
  там, где это нужно, без постройки полноценного дома.

## 5. Стартовый сценарий

В палаточной эре последовательность:

1. Группа прибывает к входной табличке.
2. Игрок строит **склад под открытым небом** — где угодно, без ограничений.
3. Игрок строит **Главный костёр 1-го уровня** — где угодно, без ограничений.
   До постройки костра в меню построек **заблокированы все здания, кроме
   склада и самого костра**.
4. С момента завершения строительства костра вокруг него образуется область
   деревни с базовым радиусом. Все остальные здания разблокируются.
5. В области костра игрок строит **временную палатку** или **жилую палатку**.
   Палатка расширяет область.
6. При необходимости игрок ставит **столбы** (2 ветки) для дополнительного
   расширения границы без постройки жилья.
7. В расширенной области строятся костёр для готовки, сборщик росы, туалет и
   т.д.

До постройки костра игрок может строить **только** склад под открытым небом
и сам костёр. Это мотивирует быстро завершить строительство костра и не
откладывать развитие лагеря.

## 6. Привязка юнитов к поселению

### 6.1. Принадлежность

Каждый юнит **принадлежит одному поселению** — тому, в котором он был
заселён (в палатке, доме и т.д.). Принадлежность определяется по ближайшему
костру на момент заселения или по явному назначению игрока.

### 6.2. Обслуживание зданий

Юнит, приписанный к поселению A:

- **Обслуживает здания только своего поселения** — работает на рабочих
  местах, выполняет приказы, обслуживает производство и логистику в пределах
  области деревни A.
- **Не идёт обслуживать здания в чужом поселении B** — даже если здание B
  ближе или имеет свободное рабочее место.
- Это правило распространяется на все типы работ: производство, кухню,
  торговлю, курьерскую доставку между складами и т.д.

### 6.3. Сборщики и выход за границы

**Сборщики** (охотники, собиратели, работники двора материалов) —
**исключение**: они могут выходить за пределы области своей деревни для
добычи ресурсов (ветки, трава, пища у леса/поляны).

- Сборщик всегда возвращается на склад **своего** поселения.
- Добытые ресурсы доставляются только в склады поселения, к которому
  приписан сборщик.
- Сборщик не может использовать склады чужого поселения.

### 6.4. Курьеры между поселениями

Курьерская доставка **между поселениями одного игрока** разрешена, если
игрок явно создаёт торговый приказ между ними. Курьер из поселения A может
отнести ресурсы на склад поселения B, но это не делает его работником
поселения B — после доставки он возвращается в своё поселение.

## 7. Архитектура реализации

### 7.1. Домен

`game/features/buildings/domain/village_territory.gd`:

- `VillageTerritory` — типизированная запись: центр (костёр), радиус костра,
  список якорей (дома и столбы) с их позициями и радиусами.
- Детерминированные функции: `is_inside(cell: Vector2i) -> bool`,
  `is_inside_foreign(cell: Vector2i) -> bool`.
- Расчёт объединённой области — множество клеток сетки.
- `campfire_limit_for_era(era: int) -> int` — лимит костров для эры.

### 7.2. Приложение

`game/features/buildings/application/village_territory_service.gd`:

- Читает `BuildingRegistry` для отслеживания построенных костров, жилых
  домов и столбов.
- Пересчитывает область при постройке, сносе, апгрейде зданий.
- Предоставляет `can_place(building_type: StringName, cell: Vector2i) -> bool`
  и `placement_reason(...) -> String` для UI.
- Проверяет лимит костров для текущей эры.
- Хранит список чужих деревень (для будущих сценариев с NPC-поселениями).
- Отслеживает принадлежность юнитов к поселениям (см. раздел 6).

### 7.3. Интеграция

- `_can_place` в `settlement_game.gd` дополняется вызовом
  `village_territory_service.can_place(...)`.
- Старая система простого радиуса от костра **полностью удаляется** из
  `_can_place` и связанных функций.
- `BuildingRegistry` эмитит сигнал при добавлении/удалении записи; сервис
  подписывается и пересчитывает область.
- `building_catalog.gd` получает флаги для каждого определения здания:
  - `requires_village_area: bool` — требует ли здание области деревни;
  - `expands_village_area: bool` — расширяет ли здание область;
  - `is_campfire: bool` — является ли здание Главным костром.
- Меню построек фильтрует список через `village_territory_service`:
  до постройки костра показываются только склад и костёр.
- Система приказов и `CitizenBrain` учитывает принадлежность юнита к
  поселению: приказы и рабочие места фильтруются по `village_id` юнита.

### 7.4. Презентация

`game/features/buildings/presentation/village_boundary_markers.gd`:

- `Node3D`, который подписывается на изменения области и обновляет
  автоматические маркеры границы.
- Маркеры — простые мешевые инстансы, не влияющие на навигацию и коллизию.
- Столбы (boundary posts) — отдельные здания с собственными мешами в
  `building_blueprints.gd`, рендерятся через стандартную систему зданий.
